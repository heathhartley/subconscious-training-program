# TrainSmart

TrainSmart is a full-stack training plan generator that combines a modern React + TypeScript frontend with an AI-powered backend. Users can describe the type of training plan they want, optionally connect Strava, and receive a personalized multi-week training schedule generated by an AI agent.

This project was built as a take-home assignment and is designed to demonstrate frontend architecture, backend integration, API design, and thoughtful tradeoffs around deployment and iteration speed.

---

## Overview

**What TrainSmart does:**

- Allows users to describe a training goal in natural language
- Generates a personalized, multi-week training plan using an AI agent
- Supports Strava integration so historical training data can inform the plan
- Displays training plans in a clean, responsive UI with week-by-week navigation

**High-level architecture:**

- React + TypeScript frontend
- FastAPI backend
- AI agent powered by the Subconscious API
- Deployed using lightweight AWS services for speed and simplicity

## Frontend

### Tech Stack

- TypeScript
- Vite
- Tailwind CSS
- Lucide Icons
- Framer Motion

### Features

- Text-based training goal input
- Preset goal shortcuts (speed, strength, weight loss, etc.)
- Horizontally scrollable week selector (mobile + desktop friendly)
- Clean, accessible UI with subtle animations
- Disabled but intentional UI for upcoming integrations (Apple Health, Garmin)

### Running Locally

To install dependencies and start the frontend locally:
``` 
npm install
npm run dev
```


### Deployment

- Deployed to **AWS S3** for quick static hosting
- Future iterations would use:
  - **CloudFront** for CDN and HTTPS
  - **Route 53** with a custom domain

The current approach prioritizes fast iteration over production-grade infrastructure.

## Design Decisions & Tradeoffs

- **S3 over CloudFront**: faster to ship for a take-home; easy to upgrade later
- **Polling instead of WebSockets**: simpler, sufficient for generation latency
- **Mocked Strava data**: keeps focus on architecture and AI flow
- **Single repo**: easier to review and reason about in an interview context
- **Strong typing (TypeScript)**: improves maintainability and confidence when rendering AI-generated data
